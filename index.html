What Are AI-Powered GitHub Actions?

These GitHub Actions workflows enable you to trigger AI coding assistants directly from GitHub issues and pull requests. Simply mention an AI agent in a comment, and the workflow automatically:

  • Analyzes the issue or pull request context
  • Reads relevant code and files from your repository
  • Generates fixes or provides comprehensive code reviews
  • Creates pull requests with the fixes or posts detailed reviews
  • Comments back with results and links to the work done
Production-Ready

These workflows are battle-tested in real projects and include security controls, permission management, and authorization checks to ensure only authorized users can trigger AI agents.

How It Works: The Complete Workflow

Fix Workflow (Write Access)

1

Trigger via Comment

An authorized user comments on an issue or pull request with a trigger phrase like @claude-fix, @codex-fix, or @cursor-fix.

2

Authorization Check

The workflow verifies that the user is in the authorized list. If not authorized, it posts a message explaining that only maintainers can trigger the workflow.

3

Context Loading

The workflow loads the issue fix prompt template, gathers GitHub context (repository, issue/PR number, triggering user), and fetches full issue/PR details including comments.

4

AI Agent Execution

The AI agent runs with write access to the repository. It:

  • Reads the issue description and all comments
  • Searches the codebase for relevant files
  • Performs root cause analysis
  • Implements a minimal, focused fix
5

Pull Request Creation

The agent creates a new branch, commits the changes, and opens a pull request with:

  • Clear title describing the fix
  • Detailed description of the root cause
  • Explanation of the changes made
  • Links back to the original issue
6

Issue Comment

The workflow posts a comment on the original issue with a link to the newly created pull request, notifying the team that a fix is ready for review.

Review Workflow (Read-Only)

1

Trigger via Comment

An authorized user comments on a pull request with a trigger phrase like @claude-review, @codex-review, or @cursor-review.

2

Authorization & Context

After authorization check, the workflow loads the PR review prompt template with GitHub context and PR details.

3

AI Code Review

The AI agent performs a comprehensive code review with read-only access, analyzing:

  • Code quality and best practices
  • Security vulnerabilities
  • Performance considerations
  • Test coverage gaps
  • Architectural concerns
4

Review Posted

The workflow posts a detailed review comment on the PR with structured feedback including critical issues, important suggestions, and minor improvements.

Permission Model

Fix Workflows (Write Access)
  • contents: write
  • pull-requests: write
  • issues: write
  • id-token: write
  • actions: read
Review Workflows (Read-Only)
  • contents: read
  • pull-requests: write (comments only)
  • issues: write (comments only)
  • id-token: write
  • actions: read
Unauthorized Handler
  • issues: write (rejection messages)
  • pull-requests: write (rejection messages)
Security Note:

The write permissions for "issues" and "pull-requests" in review workflows are limited to creating comments only. AI agents in review mode cannot modify code or create branches.

Setup Instructions

1. Copy Workflow Files

Copy all files from the .github directory to your repository:

.github/ ‚îú‚îÄ‚îÄ workflows/ ‚îÇ ‚îú‚îÄ‚îÄ claude-fix.yml ‚îÇ ‚îú‚îÄ‚îÄ claude-review.yml ‚îÇ ‚îú‚îÄ‚îÄ codex-fix.yml ‚îÇ ‚îú‚îÄ‚îÄ codex-review.yml ‚îÇ ‚îú‚îÄ‚îÄ cursor-fix.yml ‚îÇ ‚îî‚îÄ‚îÄ cursor-review.yml ‚îú‚îÄ‚îÄ issue_fix_prompt.md ‚îú‚îÄ‚îÄ pr_review_prompt.md ‚îî‚îÄ‚îÄ pull_request_template.md

2. Configure Secrets

Add the required API keys as repository secrets in GitHub Settings ‚Üí Secrets and variables ‚Üí Actions:

  • CLAUDE_CODE_OAUTH_TOKEN - OAuth token for Claude Code API
  • OPENAI_API_KEY - API key for OpenAI Codex
  • CURSOR_API_KEY - API key for Cursor AI
Important:

Never commit these API keys to your repository. Always use GitHub Secrets for sensitive credentials.

3. Authorize Users

Edit each workflow file to add authorized users who can trigger the AI agents. Look for this section:

# Only trigger for authorized users if: contains(fromJSON('["your-github-username"]'), github.event.comment.user.login)

Replace "your-github-username" with a list of authorized GitHub usernames.

4. Customize Prompts

The workflow behavior is controlled by two prompt templates:

  • issue_fix_prompt.md - Instructions for fix workflows
  • pr_review_prompt.md - Instructions for review workflows

Edit these files to customize how AI agents analyze issues and review code for your specific project needs.

5. Test the Workflows

Create a test issue and try triggering an AI agent:

  1. Create a GitHub issue describing a bug or improvement
  2. Comment on the issue with @claude-fix
  3. Watch the Actions tab to see the workflow run
  4. Review the resulting pull request

Usage Examples

Triggering a Fix

When you encounter a bug in an issue:

# Issue #42: Button click handler not working The submit button doesn't trigger the form submission. Steps to reproduce: 1. Fill out the form 2. Click submit button 3. Nothing happens # Comment to trigger fix: @claude-fix Please investigate and fix this button issue

What happens next:

  1. Claude analyzes the issue and searches for the button code
  2. Identifies the missing event listener
  3. Implements the fix
  4. Creates a PR with the solution
  5. Comments on the issue with the PR link

Requesting a Code Review

On a pull request that needs review:

# PR #123: Add user authentication This PR implements JWT-based authentication for the API. # Comment to request review: @claude-review Please review this authentication implementation

What happens next:

  1. Claude reads all changed files
  2. Analyzes security implications
  3. Checks for best practices
  4. Reviews error handling and edge cases
  5. Posts a detailed review comment with findings

Using Multiple Agents

You can trigger multiple agents for different perspectives:

# Issue #56: Performance issue with data processing The data processing function is taking too long for large datasets. # Get multiple perspectives: @claude-fix Could you analyze and optimize this? @codex-fix Alternative approach? # Or get multiple reviews: @claude-review Focus on architectural concerns @codex-review Focus on performance optimizations

Benefits of multiple agents:

  • Different AI models may propose different solutions
  • Compare approaches and choose the best one
  • Get specialized feedback (architecture vs. performance)
  • Learn from diverse perspectives

Best Practices

When to Use Fix Workflows

  • Clear, reproducible bugs - Issues with specific error messages and steps to reproduce
  • Well-scoped improvements - Adding a specific feature or refactoring a particular component
  • Routine maintenance - Updating dependencies, fixing linting issues, adding tests
  • Quick wins - Simple improvements that don't require product decisions

When to Use Review Workflows

  • Security-sensitive changes - Authentication, authorization, data handling
  • Complex features - Large PRs that need thorough analysis
  • Architectural changes - Modifications to core systems or patterns
  • Learning opportunities - Get detailed feedback on your implementation approach

Tips for Better Results

  • Provide clear, detailed issue descriptions with examples
  • Include error messages, stack traces, and logs when applicable
  • Mention specific files or functions that might be relevant
  • Add context about the expected vs. actual behavior
  • Review AI-generated code before merging - always validate the fix
  • Use descriptive trigger comments to guide the AI's focus
Remember:

AI agents are powerful assistants, but they're not perfect. Always review their work, run tests, and validate that fixes actually solve the problem before merging to production.

Troubleshooting

Workflow Not Triggering

  • Verify you're using the exact trigger phrase (e.g., @claude-fix)
  • Check that your GitHub username is in the authorized users list
  • Ensure the workflow file is in .github/workflows/
  • Check the Actions tab for any error messages

Unauthorized Error

  • Your username must be in the fromJSON() array in the workflow
  • Check the exact spelling of your GitHub username
  • Ask a repository maintainer to add you to the authorized users list

AI Agent Failed to Create PR

  • Check if API keys are correctly configured in repository secrets
  • Verify the AI agent had sufficient context to understand the issue
  • Review the workflow logs in the Actions tab for specific errors
  • The issue might require human judgment or product decisions

Review Posted But Incomplete

  • Large PRs may exceed token limits - consider breaking them up
  • Complex architectural changes might need human review
  • Check the workflow timeout settings (default 15-30 minutes)

Architecture Deep Dive

Workflow Structure

Each workflow consists of three main components:

1. Event Triggers

on: issue_comment: types: [created] pull_request_review_comment: types: [created]

Workflows trigger when comments are created on issues or pull requests.

2. Authorization Job

if: | contains(github.event.comment.body, '@claude-fix') && contains(fromJSON('["authorized-user"]'), github.event.comment.user.login)

Checks that the trigger phrase is present and the user is authorized.

3. AI Agent Execution

  • Context Loading: Fetches issue/PR details and loads prompt templates
  • Agent Invocation: Runs the AI agent with custom instructions
  • Result Handling: Creates PR or posts review comment
  • Notification: Updates the original issue/PR with results

Prompt Engineering

The effectiveness of AI agents depends heavily on well-crafted prompts. The templates include:

  • Role Definition: Clear explanation of the agent's purpose
  • Architecture Context: Information about the project structure
  • Workflow Steps: Step-by-step instructions for the agent
  • Decision Points: Guidance on when to proceed vs. ask for help
  • Output Format: Structured format for reviews and PR descriptions

Security Considerations

  • Least Privilege: Review workflows have read-only access to repository contents
  • Authorization Lists: Only specific users can trigger workflows
  • Secret Management: API keys stored as encrypted GitHub Secrets
  • Audit Trail: All workflow runs are logged in the Actions tab
  • Branch Protection: AI-created PRs still require human review before merging

Advanced Customization

Modifying Prompt Templates

Customize issue_fix_prompt.md and pr_review_prompt.md to:

  • Add project-specific context and architecture documentation
  • Include coding standards and style guidelines
  • Define custom review criteria and focus areas
  • Add links to relevant documentation
  • Specify testing requirements

Adjusting Timeout Settings

timeout-minutes: 30 # Fix workflows timeout-minutes: 15 # Review workflows

Increase for complex codebases or decrease to fail faster.

Tool Restrictions

Limit which tools the AI agent can use by uncommenting and modifying the allowed_tools parameter:

# Example: Read-only tools for reviews allowed_tools: "Read(*),Grep(*),Bash(git log*),Bash(git diff*)" # Example: Full access for fixes allowed_tools: "Edit(*),Write(*),Read(*),Grep(*),Bash(git *)"

Adding More AI Agents

To add a new AI agent:

  1. Copy an existing workflow file (e.g., claude-fix.yml)
  2. Rename it (e.g., gemini-fix.yml)
  3. Update the trigger phrase and agent action
  4. Add required API keys to repository secrets
  5. Update the authorization list

Cost Considerations

API Usage

Each workflow run consumes API credits from the respective AI service:

  • Claude Code: Charged per token (input + output)
  • OpenAI Codex: Charged per API call and token usage
  • Cursor AI: Subscription-based or per-usage pricing

GitHub Actions Minutes

Workflows run on GitHub Actions runners, consuming action minutes:

  • Public repositories: Unlimited minutes
  • Private repositories: Limited by plan (2,000-50,000 minutes/month)
  • Self-hosted runners: No minute costs

Optimization Tips

  • Use review workflows more liberally (faster, cheaper)
  • Reserve fix workflows for well-scoped issues
  • Set appropriate timeout limits to avoid long-running jobs
  • Monitor API usage and set up billing alerts
  • Consider using smaller/faster models for simple tasks
